App Store Connect + TestFlight — 操作检查清单
===============================================
项目: Surrogate Agency USA (My Surro)
Bundle ID: io.dcloud.mysurronew
当前 app 版本 (expo.version): 1.7.6
说明: 以下步骤须在浏览器登录 App Store Connect 完成；本文件与仓库内模板配合使用。

────────────────────────────────────────
[ ] 1. Export Compliance（对应 TestFlight 灰掉 / 无法分配 build）
────────────────────────────────────────
路径: App Store Connect → 你的 App → TestFlight → iOS Builds
1) 打开 build 1.7.6 (7)（或当前最新 Complete 的 build）。
2) 若显示 “Missing Compliance” 或黄色提示：
   - 点击进入合规问答。
   - 与工程声明一致：app.json 中 ITSAppUsesNonExemptEncryption = false
     通常选「App 仅使用豁免加密（HTTPS 等标准加密）」类选项；以页面英文/中文表述为准。
3) 保存后刷新；合规未完成时 Add Group / 分配测试员常会灰掉。

────────────────────────────────────────
[ ] 2. Internal Testing（内部测试）
────────────────────────────────────────
路径 A — 加人:
  Users and Access → People → + 邀请用户
  - 使用对方 Apple ID 邮箱；对方必须接受邀请邮件。
  - 角色 Developer 通常足够参与内部测试（以你团队策略为准）。

路径 B — 把 build 分给组:
  TestFlight → Internal Testing → App Store Connect Users
  - Builds: 确认已包含 1.7.6 (7)。
  - Testers: 添加已接受的内部用户。

若 “Add Group” 弹窗里复选框灰、Next 不可用:
  - 先确认该组是否已关联此 build（Group(1) 已显示时，可能无需再 Add）。
  - 先完成步骤 1 Export Compliance。
  - 换浏览器无痕窗口或重新登录。

What to Test:
  - 复制 submission/WHAT_TO_TEST_EN.txt（或自行改写）→ 粘贴保存。

────────────────────────────────────────
[ ] 3. App Review Information（审核用登录账号）
────────────────────────────────────────
路径: App Store Connect → App → App Store（分发）→ 准备提交的版本
  → App Review Information → Sign-in required（若应用必须登录）

填写内容: 见 submission/APP_REVIEW_SIGNIN_TEMPLATE.txt
  - 替换占位符为真实测试账号与密码。
  - 说明如何从启动到核心界面（Parents / Surrogate 各一条路径）。

────────────────────────────────────────
[ ] 4. 正式上架 App Store（公开下载）
────────────────────────────────────────
前置: Agreements, Tax, and Banking 无待办（含付费协议若涉及内购/付费 App）。

路径: App Store Connect → App → Distribution / App Store
1) 创建或打开版本 1.7.6（与 expo.version 一致或按你上架策略调整）。
2) Build: 选择已处理完成的构建（如 1.7.6 (7) 或更新 build）。
3) 必填: 截图、描述、关键词、支持 URL、隐私政策 URL、App Privacy 问卷、分级等。
4) 确认 App Review Information 与加密/出口合规一致。
5) Submit for Review。

────────────────────────────────────────
完成全部 [ ] 后，应用进入「等待审核」状态。
